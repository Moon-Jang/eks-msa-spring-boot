## 도커 이미지 빌드 및 ECR 푸시

### 이미지 빌드

- 빌드 대상 EKS 노드가 `t3.small`(x86_64/amd64)인데 로컬이 Apple Silicon(arm64)인 경우 `--platform linux/amd64`를 반드시 지정해야 함
- 컨텍스트 경로(`.`)를 빼먹으면 `docker buildx build requires 1 argument` 에러 발생

```Shell
docker build --platform linux/amd64 -t 831926604059.dkr.ecr.ap-northeast-2.amazonaws.com/ordersystem-backend:latest .
```

### ECR 로그인

```Shell
aws-vault exec moonjang -- aws ecr get-login-password --region ap-northeast-2 | docker login --username AWS --password-stdin 831926604059.dkr.ecr.ap-northeast-2.amazonaws.com
```

- ECR 로그인 URL은 리포지토리 경로(`/ordersystem-backend`) 없이 **레지스트리 도메인까지만** 사용

### 이미지 푸시

```Shell
docker push 831926604059.dkr.ecr.ap-northeast-2.amazonaws.com/ordersystem-backend:latest
```

### 트러블슈팅

1. **`openjdk:17-jdk-alpine: not found`**
   - Docker Hub 공식 `openjdk` 이미지가 deprecated되어 삭제됨
   - `eclipse-temurin:17-jdk-alpine`(빌드 스테이지) / `eclipse-temurin:17-jre-alpine`(런타임 스테이지)으로 대체

2. **`no match for platform in manifest: not found`**
   - 로컬(arm64)과 배포 대상(amd64) 플랫폼 불일치
   - `--platform linux/amd64` 옵션으로 해결

3. **`error saving credentials: ... already exists in the keychain (-25299)`**
   - macOS Keychain에 예전 ECR 로그인 정보가 손상된 상태로 남아있어 발생
   - 확인: `security find-internet-password -s <ECR 레지스트리 도메인>`
   - 삭제: `security delete-internet-password -s <ECR 레지스트리 도메인> -a AWS`
   - 삭제 후 다시 `docker login` 실행하면 정상 저장됨

4. **`Credentials were refreshed, but the refreshed credentials are still expired` (ECR 로그인 시)**
   - aws-vault 서브쉘 안의 자격증명 환경변수가 만료된 채로 굳어버린 경우 발생 (환경변수는 자동 갱신되지 않음)
   - 해결: 서브쉘 `exit` 후 `aws-vault exec <profile> --duration=12h -- zsh`로 재인증
