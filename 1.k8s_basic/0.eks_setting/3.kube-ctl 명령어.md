## kubectl 로컬 세팅

1. 설치

```Shell
brew install kubectl
```

2. 설치 확인

```Shell
kubectl version --client
```

3. kube config세팅

```Shell
aws-vault exec <본인의_프로필_이름> -- aws eks update-kubeconfig --region ap-northeast-2 --name test_cluster
```

- 이 명령어는 AWS CLI의 일부로 EKS클러스터와 상호 작용하기 위한 kubectl 구성 파일(kubeconfig)을 자동으로 생성하거나 업데이트
- 이 명령어를 사용하면 수동으로 구성 파일을 생성하고 관리할 필요 없이, 쉽게 AWS EKS 클러스터와의 연결을 설정

[결과]

```
Added new context arn:aws:eks:ap-northeast-2:~~:cluster/eks-practice to ~/.kube/config
```

[세팅 확인]
```Shell
cat ~/.kube/config
```

[클러스터 전환]
```Shell
aws eks update-kubeconfig --region ap-northeast-2 --name test-cluster
```

4. 워커 노드 확인
```Shell
kubectl get nodes
```

## 네임스페이스 생성

- 애플리케이션 매니페스트(redis.yml, ordersystem 등)가 `mj-eks` 네임스페이스를 참조하므로, 배포 전에 미리 생성해둬야 함
- 클러스터를 새로 만들 때 기본 제공되는 네임스페이스(`default`, `kube-system` 등)에는 포함되어 있지 않아 누락하기 쉬움

```Shell
kubectl create namespace mj-eks
```

[생성 확인]
```Shell
kubectl get ns
```
