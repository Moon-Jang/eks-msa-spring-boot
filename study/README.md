# 스터디 문제 은행

인프런 EKS 강의 스터디용 문제 출제/채점 워크플로우.

## 구조

```
study/
├── problems/
│   ├── villain/<날짜>-<주제>/     # 빌런 모드 (YAML에 버그 심기)
│   │   ├── problem.yaml           # 풀이자에게 공유하는 버그 매니페스트
│   │   ├── README.md              # 시나리오 설명 (버그 위치 언급 없음)
│   │   └── answer-key.md          # 정답 (gitignore, 커밋 금지)
│   └── scenario/<날짜>-<주제>/    # 실무 시나리오 문제
│       ├── problem.md             # 풀이자에게 공유하는 시나리오
│       └── answer-key.md          # 채점 루브릭 (gitignore, 커밋 금지)
└── submissions/                   # (선택) 풀이자 제출 내용 보관용
```

## 사용법

1. **문제 출제** (아무나 한 명이 담당)
   - `/k8s-villain-mode <주제>` — 버그 심은 매니페스트 문제 생성
   - `/k8s-practice-scenario <주제>` — 실무 의사결정/트러블슈팅 시나리오 문제 생성
   - 생성된 `problem.yaml`/`README.md` 또는 `problem.md`만 다른 스터디원에게 공유한다.
     `answer-key.md`는 로컬에만 있고 git에 올라가지 않는다.
2. **풀이**: 다른 스터디원이 문제를 풀고 수정한 매니페스트/명령/서술형 답변을 작성한다.
3. **채점**: 출제자가 `k8s-answer-checker` 에이전트에게 문제 폴더 경로 + 풀이자의 제출 내용을 전달한다.
   에이전트가 answer-key.md와 대조해 통과 여부, 놓친 포인트, 후속 질문을 정리해준다.

## 원칙

- 3번(실무 시나리오)은 CKA 암기형 문제가 아니라, 이 레포/강의 맥락에서 실제로 마주칠 법한
  설계·트러블슈팅 문제를 지향한다.
- answer-key.md는 절대 커밋하지 않는다 (스포일러 방지).
