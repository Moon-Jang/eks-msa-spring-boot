# 답변

## 1. VPC/서브넷 설계

서브넷이 퍼블릭으로 되어있으면 네트워크가 열려있기때문에 보안의 취약하다 누군가 네트워크에 침입해 노드내에 파일이나 설정 같은 탈취하거나 악성코드를 심을 수 잇음.

개선방향: 모두 프라이빗 서브넷으로 바꿔야한다. 프라이빗 서브넷으로 되어있어도 인그레스 컨트롤러를 쓰기 때문에 외부 요청을 클러스터 내부로 유입할 수 있고 인그레스 컨트롤러는 alb이기 때문에 보안그룹 세팅이 가능

## 2. 보안그룹

RDS는 외부로 오픈되면 안되고 철저히 격리된 환경에 있어야된다. 어차피 같은 VPC를 공유하기 때문에 서브넷의 privateIP CIDR 을 확인하고 해당 ip의 3306을 열어주면 된다.

# 채점 결과

## 1. VPC/서브넷 설계 — 통과 (핵심 포인트 대부분 반영)
- 퍼블릭 서브넷 리스크를 "노드 침해 → 파일/설정 탈취, 악성코드 심음"으로 구체적으로 짚은 점 좋음
- 프라이빗 서브넷 전환 + ALB Ingress로 외부 요청을 계속 받을 수 있다는 것까지 언급한 건 핵심 포인트 정확히 커버
- **누락**: 문제에서 명시한 "예산 제약상 최소 변경" 조건에 대한 절충안이 빠짐 — 전면 프라이빗 전환은 NAT Gateway 비용이 추가로 드는데, 이 트레이드오프 언급이 없음. 또한 퍼블릭 서브넷에서도 워커노드 자체는 보안그룹으로 인바운드를 원천 차단하면 즉시 완화 가능하다는 "최소 변경" 대안도 없음

## 2. 보안그룹 — 부분 통과, 핵심 포인트 하나 놓침
- "외부로 오픈되면 안 된다"는 방향은 맞고, 같은 VPC 내부에서만 허용해야 한다는 것도 맞음
- **중요 누락**: "서브넷의 privateIP CIDR을 확인해서 3306 열기"는 CIDR 기반 접근이라 answer-key가 요구하는 핵심 — **Security Group ID를 소스로 지정**(CIDR이 아니라)하는 방식보다 한 단계 아래임. 노드 IP는 오토스케일링/재생성으로 바뀔 수 있는데, CIDR로 열면 서브넷 전체가 열리거나 IP 변경 시 규칙이 깨질 위험이 있음. "워커 노드 보안그룹을 소스로 지정"이 정석

## 3. IAM — 미응답 (개념 설명)
`withOIDC: false`가 뭔지 모른다고 했으니 개념 정리:
- **OIDC(OpenID Connect)**: EKS 클러스터에는 자체 OIDC 발급자(issuer)가 있는데, `withOIDC: true`로 켜면 이걸 AWS IAM이 신뢰할 수 있는 자격증명 발급자로 등록함
- **IRSA (IAM Roles for Service Accounts)**: 이 OIDC 연동이 있어야, 쿠버네티스의 **서비스어카운트(ServiceAccount) 단위로 IAM Role을 매핑**할 수 있음. 예를 들어 "ordersystem 파드는 특정 S3 버킷만 read 가능한 역할"을 그 파드의 서비스어카운트에만 부여 가능
- **`withOIDC: false`(현재 상태)의 문제점**: 이 매핑이 불가능해서, AWS 리소스 접근 권한을 파드 단위가 아니라 **노드 전체가 공유하는 노드 IAM 역할**(`AmazonEKSNodeRole`)에 줘야 함. 같은 노드에 떠 있는 모든 파드가 사실상 그 권한을 공유하게 됨 — 최소 권한 원칙 위배
- 비유: OIDC/IRSA 없이는 "건물(노드) 마스터키를 모든 입주자(파드)에게 나눠주는 것"이고, IRSA를 쓰면 "각 사무실(파드)마다 그 방만 여는 개별 키를 발급하는 것"

## 4. 외부 관리형 DB vs RDS — 통과
- "완벽한 격리 안 됨", "관리 포인트 증가"라는 핵심 리스크 정확히 짚음
- VPC 피어링/같은 VPC 통신으로의 개선 방향도 맞음
- 다만 answer-key가 요구하는 **"각각 언제 정당화되는지"**(양방향 트레이드오프)는 살짝 부족 — RDS 단점 없이 "RDS가 낫다"는 방향으로만 서술됨. 외부 관리형이 유리한 경우(엔진 버전, 스터디/PoC 등)도 같이 생각해보면 좋음

## 종합
**부분 통과, 재학습 권장 항목: 3번(IAM/IRSA)**. 1, 2, 4번은 방향은 맞으나 세부 정확도(보안그룹은 SG 참조 방식, 트레이드오프 서술)를 조금 더 다듬으면 좋음.
